VERSIONING RULES:
    Synchronicity v[W.X.Y.Z](greek letter)
        v : decoration -> stands for "version"

        W : major version/revision/rework
        X : major features
        Y : minor features/major bug fix
        Z : minor changes/bug fixes

        greek letter : denotes software/branch major state
            α => alpha   -> early production, main features not fully implemented and unsuitable for proper play
            β => beta    -> pre-release, main major features functional, may lack some minor features
            χ => release -> full release, ready for roll-out and distribution
            ν => nightly -> early unstable feature complete test builds, may contain new test minor features and/or fixes
